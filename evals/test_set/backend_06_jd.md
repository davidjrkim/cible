# Backend Engineer — Supabase

**Company:** Supabase
**Location:** Fully remote (we're a fully remote, open-source company)

## About Supabase

We're the open-source Firebase alternative built on Postgres. The whole platform is open source on GitHub.

## What you'll do

- Work on Realtime — our WebSocket-based Postgres change-feed service (Elixir/Phoenix)
- Or work on PostgREST and Auth — both forks we contribute heavily upstream to
- Improve Postgres extensions we ship (we maintain `pg_graphql`, `pgvector` packaging, `pgsodium`, etc.)
- Be active on GitHub — issue triage, discussions, and PRs from the community are part of the job

## What we want

- 4+ years backend experience
- Deep Postgres knowledge — you can explain MVCC, write a function in PL/pgSQL, and read `EXPLAIN ANALYZE`
- Comfort with Elixir/OTP **or** Rust **or** Go (we have all three across the company)
- You like working in the open

## Bonus

- You've written or maintained a Postgres extension (in C, pgrx, or as an SQL-only extension)
- You've operated Postgres replication in production
