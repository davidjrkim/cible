# Senior Backend Engineer — Fly.io

**Company:** Fly.io
**Location:** Remote

## About Fly

Fly runs full-stack apps and databases close to users, all over the world. Our infrastructure is Firecracker VMs, our own routing fabric, and a lot of Rust and Elixir.

## What you'd do

- Work on the Machines API — the control plane that orchestrates customer Firecracker VMs across our ~30 regions
- Write production Elixir (the API control plane) and Rust (the on-host agent that manages VMs)
- Investigate weird networking issues (you will become friends with `tcpdump` and `bpftrace`)
- On-call rotation, ~1 week in 8. Fly's on-call is real; the public status page is, too.

## What we want

- 5+ years of backend engineering experience
- You've operated a distributed system in production (not just consumed AWS managed services)
- Comfort with at least one of: Elixir / OTP, Rust, or Go
- You're patient with Linux networking. eBPF, iptables, namespaces, all the joy.

## What's the work like?

You'll ship code daily, write internal RFCs, and post in #incidents when you break things. We have a public engineering blog and we expect engineers to write for it.
